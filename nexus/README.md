# nexus/

This folder contains canonical NEXUS knowledge content.

It is intended to hold the human-readable and machine-loadable definitions that describe the current ontology, laws, lenses, and examples.

## Subfolders

- `core/` — base ontology and IDs
- `definitions/` — canonical concept definitions
- `laws/` — invariant rules and constraints
- `lenses/` — descriptions of modeling lenses
- `examples/` — worked examples

## Rule

Content here is canonical unless explicitly marked as generated or experimental.

## Why this folder exists

This repo separates:

- `nexus/` = what the system means
- `src/` = code that works with that meaning

That separation keeps the source of truth clear.
